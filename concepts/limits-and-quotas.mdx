KWALA provides flexible usage limits designed to support developers from early-stage projects to enterprise-scale automation. This page outlines the limits and quotas for each plan.

## Starter plan

The Starter plan is designed for solo developers, small teams, and early-stage projects.

**Price**: $49/month (includes fixed infrastructure cost for non-transactional operations)

### Included limits

| Feature | Limit |
|---------|-------|
| **Block triggers** | 10,000 per month |
| **Event triggers** | Unlimited |
| **Price triggers** | Unlimited |
| **Time triggers** | Unlimited |
| **Web2 API calls** | Unlimited |
| **Actions per workflow** | Up to 10 |

### Pay-per-use

The following operations are billed based on actual gas consumption:

- **Web3 transactions**: Pay-per-gas
- **Smart contract deployments**: Pay-per-gas

## Custom plan

The Custom plan is built for scale, enterprise automation, or bespoke integrations.

**Price**: Custom pricing tailored to your infrastructure and credit needs

### Everything in Starter, plus

- Volume-based discounting on trigger types
- Dedicated workflow nodes or regions
- SLA-backed uptime guarantees
- Custom compliance filters (KYC/allowlists)
- White-glove onboarding and workflow design

## Workflow limits

| Limit | Value |
|-------|-------|
| **Actions per workflow** | 10 |
| **Workflows per account** | Unlimited |
| **Concurrent active workflows** | Based on plan |

## Credit consumption

KWALA credits are consumed for various operations:

- **Trigger monitoring**: Credits are deducted for monitoring blockchain events and conditions
- **Workflow activation**: Deploying and activating workflows requires credits
- **Action execution**: Each action execution consumes credits based on complexity

<Info>
  Failed workflows consume minimal credits for the processing attempt, but not the full execution cost.
</Info>
